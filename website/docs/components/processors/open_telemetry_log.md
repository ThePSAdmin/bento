This processor sends log records to an OpenTelemetry collector using the OTLP protocol.
All fields support Bloblang interpolation, allowing dynamic values based on message content.

The processor follows the [OpenTelemetry Logs Data Model](https://opentelemetry.io/docs/specs/otel/logs/data-model/) specification.


## Fields

### `http`

A list of HTTP collectors.


Type: `array`  
Default: `[]`  

### `http[].address`

The endpoint of a collector to send log events to.


Type: `string`  

```yml
# Examples

address: localhost:4318
```

### `http[].secure`

Connect to the collector over HTTPS.


Type: `bool`  
Default: `false`  

### `grpc`

A list of gRPC collectors.


Type: `array`  
Default: `[]`  

### `grpc[].address`

The endpoint of a collector to send log events to.


Type: `string`  

```yml
# Examples

address: localhost:4317
```

### `grpc[].secure`

Connect to the collector with client transport security.


Type: `bool`  
Default: `false`  

### `body`

The body/message of the log record. This field is required and supports interpolation.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  

```yml
# Examples

body: ${! json() }

body: ${! content() }

body: User ${! json("user.name") } performed action ${! json("action") }
```

### `timestamp`

The timestamp when the event occurred. Supports RFC3339 format or Unix timestamp (seconds or nanoseconds). If not set, the current time is used.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  
Default: `""`  

```yml
# Examples

timestamp: ${! json("event_time") }

timestamp: ${! metadata("timestamp") }
```

### `observed_timestamp`

The timestamp when the log record was observed. If not set, the current time is used.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  
Default: `""`  

### `severity`

The severity level of the log record. Supported values: TRACE, DEBUG, INFO, WARN, ERROR, FATAL.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  
Default: `""`  

```yml
# Examples

severity: ${! json("level") }

severity: INFO
```

### `attributes_mapping`

A Bloblang mapping that produces a flat object of key-value pairs to attach as attributes to the log record.


Type: `string`  

```yml
# Examples

attributes_mapping: |-
  root.service = "my-service"
  root.env = meta("env")
  root.user_id = this.user.id
```

### `trace_id`

A trace ID to associate with this log record for distributed tracing correlation. Should be a 32-character hex string.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  
Default: `""`  

```yml
# Examples

trace_id: ${! metadata("traceparent").split("-").index(1) }
```

### `span_id`

A span ID to associate with this log record. Should be a 16-character hex string.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  
Default: `""`  

```yml
# Examples

span_id: ${! metadata("traceparent").split("-").index(2) }
```

### `flush_timeout`

The maximum time to wait for pending log records to be exported when closing the processor.


Type: `string`  
Default: `"5s"`
